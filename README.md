# Дипломный проект: Инженер по тестированию расширенный. Практический блок: вторая часть

## Описание проекта
Проект состоит из двух функциональных частей: Работа с базой данных и автоматизированные тесты для проверки API учебного сервиса заказа самокатов.

## Структура проекта
- **database** - каталог для функциональной части связанной с базой данных
  - **task_1.sql** - скрипт вывода списка логинов курьеров с количеством их заказов в статусе "В доставке"
  - **task_2.sql** - скрипт всех трекеров и их статусы доставок
- **configuration.py** - содержит URL сервиса и пути к эндпоинтам
- **data.py** - содержит тестовые данные (тела запросов)
- **order_api.py** - функции для взаимодействия с API (создание, получение заказа)
- **test_orders.py** - автотест, проверяющий сценарий создания и получения заказа
- **README.md** - документация проекта

## Технологии
- **Python 3.7+**
  -  **pytest** – библиотека для написания и запуска тестов
  -  **requests** – библиотека  для выполнения HTTP-запросов
- **PostgreSQL** – база данных проекта
  -  **psql** - для подключения к базе данных

Убедитесь, что на вашем компьютере установлены:

- pip (менеджер пакетов Python)

## Работа с базой данных
### Клонирование репозитория
```bash
git clone <url-репозитория>
cd <имя-папки-проекта>
```
### Подключение к БД
Подключитесь к целевому серверу исходя из пререквизитов на странице обучения.
перейдите в папку database
```bash
cd database
```
Для доступа к базе данных используйте команду:
```bash
psql -U morty -d scooter_rent
```
Пароль вы можете узнать на странице обучения.
### Задание 1.
Требуется проверить отображается ли созданных заказ в базе данных.
Для этого: выведи список логинов курьеров с количеством их заказов в статусе «В доставке» (поле inDelivery = true). 

#### Выполнение:
```bash
psql \i task_1.sql
```
 
### Задание 2.
Требуется проверить корректность записи статусов заказов.
Для этого: выведи все трекеры заказов и их статусы. 
Статусы определяются по следующему правилу:
Если поле finished == true, то вывести статус 2.
Если поле canсelled == true, то вывести статус -1.
Если поле inDelivery == true, то вывести статус 1.
Для остальных случаев вывести 0.

#### Выполнение:
```bash
psql \i task_2.sql
```

## Автоматизированные тесты
###  Описание 
Данный проект представляет собой автоматизированные тесты для проверки API учебного сервиса заказа самокатов. Основной сценарий:
1. Создание заказа через POST-запрос.
2. Сохранение трекера заказа из ответа.
3. Получение данных о заказе по трекеру через GET-запрос.
4. Проверка, что код ответа равен 200.
### Клонирование репозитория
```bash
git clone <url-репозитория>
cd <имя-папки-проекта>
```
### Установка зависимостей
```bash
pip install requests pytest
```
### Настройка конфигурации
В файле configuration.py укажите актуальный URL сервиса:
```
python
URL_SERVICE = "https://<актуальный-адрес-сервера>"
```
### Запуск тестов
Запуск всех тестов
```bash
pytest test_orders.py -v
```
Запуск с подробным выводом
```bash
pytest test_orders.py -v -s
```
### Ожидаемый результат:
 - Тест должен успешно выполниться (зелёный PASSED).
 - В случае ошибки будет выведено сообщение с деталями (например, если стенд недоступен или изменилась структура API).

### Важные замечания
- Перед запуском тестов убедитесь, что сервис доступен
- Изменение тестовых данных: отредактируйте словарь default_order в файле data.py. Убедитесь, что все поля соответствуют требованиям API.
- Проверка других сценариев: вы можете добавить новые тесты в test_orders.py, импортируя функции из order_api.py и данные из data.py.
- Документация API: исходная документация доступна по адресу {URL_SERVICE}/docs/ (замените {URL_SERVICE} на базовый URL из configuration.py). Там можно уточнить форматы запросов и ответов.
- В случае ошибок проверьте актуальность URL в configuration.py
